## 관리자는 어디서 무엇을 관리하나요?

주소창에 `/admin` 을 입력하면 관리자 전용 페이지로 이동합니다.  
Firebase 이메일/비밀번호로 로그인 후 사용할 수 있습니다.

**문의 관리**
- 홈페이지에서 접수된 도입 문의 목록을 최신순으로 확인
- 각 문의를 클릭하면 상세 내용이 펼쳐짐
- 처리 상태를 3단계로 구분: 🟡 대기중 → 🟢 답변완료 → ⚫ 종료

**SNS 포스트 관리**
- 플랫폼(인스타그램·유튜브·블로그·카카오·X) 선택
- 포스트 제목, 링크, 썸네일 이미지 URL 입력 후 등록
- 등록 즉시 홈페이지 SNS 허브 섹션에 반영
- 불필요한 포스트는 목록에서 삭제 가능

## 처음 실행하려면 어떻게 하나요?

### 1단계 – 준비물 확인
- Node.js 18 버전 이상 설치 필요

### 2단계 – 설치

```bash
npm install
```

### 3단계 – 환경 변수 파일 만들기

프로젝트 루트에 `.env.local` 파일을 만들고 아래 내용을 채워 넣습니다.  
Firebase 콘솔에서 웹 앱을 추가하면 값을 얻을 수 있습니다.

```
VITE_FIREBASE_API_KEY=여기에_입력
VITE_FIREBASE_AUTH_DOMAIN=여기에_입력
VITE_FIREBASE_PROJECT_ID=여기에_입력
VITE_FIREBASE_STORAGE_BUCKET=여기에_입력
VITE_FIREBASE_MESSAGING_SENDER_ID=여기에_입력
VITE_FIREBASE_APP_ID=여기에_입력
```

> 환경 변수 파일이 없어도 홈페이지 화면은 정상 표시됩니다.  
> 단, 문의 제출과 관리자 로그인 기능은 동작하지 않습니다.

### 4단계 – 개발 서버 실행

```bash
npm run dev
```

브라우저에서 `http://localhost:5173` 으로 접속합니다.

## 배포는 어떻게 하나요?

[Vercel](https://vercel.com) 서비스를 이용합니다.

```bash
npm run build       # 배포용 파일 생성
vercel --prod       # Vercel에 배포
```

Vercel 대시보드 → **설정 → 환경 변수** 메뉴에서  
`.env.local`에 있는 Firebase 값들을 동일하게 등록해야 합니다.
